* assert: Minimal assertion utilities for writing tests and invariants.[1]
* async_hooks: Track lifecycle of async resources for context propagation and debugging.[2]
* buffer: Work with binary data using the Buffer class.[1]
* child_process: Spawn subprocesses and communicate via stdin/stdout/stderr.[1]
* cluster: Run multiple worker processes sharing a single server port.[1]
* console: Standard console logging API wired to stdout/stderr.[2]
* crypto: Cryptography primitives like hashes, HMAC, ciphers, signatures, and key management.[1]
* dgram: UDP datagram sockets for connectionless networking.[1]
* dns: DNS resolution (lookup, resolve, reverse) with system or pure JS resolvers.[1]
* domain (deprecated): Legacy unhandled error scoping; avoid in new code.[1]
* events: EventEmitter pattern for pub/sub in Node and user code.[3]
* fs: File system operations (sync, callback, stream) for files and directories.[1]
* fs/promises: Promise‑based file system API for async/await ergonomics.[2]
* http: Create HTTP servers/clients and handle requests/responses.[4]
* http2: HTTP/2 support with multiplexing and server push.[2]
* https: TLS‑secured HTTP servers/clients.[1]
* inspector: Programmatic access to V8 inspector and debugging protocol.[2]
* module: Internals/utilities for the module system; exposes builtinModules.[5]
* net: TCP and IPC sockets for stream‑based networking.[3]
* os: Operating system info (CPUs, memory, platform, tmpdir, etc.).[3]
* path: Utilities for working with file paths and separators.[3]
* perf_hooks: Performance timing, marks, measures, and observers.[2]
* process: Process info/control (argv, env, cwd, exit, nextTick).[2]
* punycode (legacy): IDNA ASCII/Unicode conversions; use WHATWG URL when possible.[1]
* querystring (legacy): Parse/format URL query strings; prefer URLSearchParams.[1]
* readline: Read lines from readable streams (e.g., stdin) interactively.[1]
* readline/promises: Promise‑based readline API for async/await flows.[2]
* repl: Read-eval-print loop for interactive Node execution.[2]
* stream: Base stream primitives (Readable, Writable, Duplex, Transform, pipeline).[3]
* stream/consumers: Helpers to consume streams into buffers, strings, JSON, or arrays.[2]
* stream/promises: Promise‑wrapped stream utilities (pipeline, finished).[2]
* string_decoder: Decode Buffer chunks into strings without breaking multibyte chars.[1]
* timers: Scheduling functions (setTimeout, setInterval, setImmediate).[1]
* timers/promises: Promise‑based timers with async/await.[2]
* tls: Secure TLS/SSL sockets and servers.[1]
* tty: TTY/terminal utilities and raw mode control.[1]
* url: WHATWG URL class and utilities for URL parsing/manipulation.[3]
* util: General utilities (promisify, callbackify, types, format, inspect).[3]
* util/types: Type‑checking helpers for specific built‑ins.[2]
* v8: Access V8 engine statistics and heap snapshots.[1]
* vm: Run code in sandboxes or separate contexts.[1]
* worker_threads: Lightweight threads sharing memory via ArrayBuffer/Atomics.[2]
* zlib: Compression/decompression (gzip, deflate, brotli streams).[1]
Notes:
* Prefer the node: prefix to force built‑in resolution, e.g., require('node:fs').[2]
